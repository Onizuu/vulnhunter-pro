\section{Introduction et Langages utilisés}

% --- Boîte Introduction (Style Rouge / Sécurité) ---
\begin{tcolorbox}[
    colback=red!5!white, 
    colframe=red!75!black, 
    title=\textbf{Introduction}, 
    fonttitle=\large\bfseries,
    rounded corners
]
    La sécurité applicative est un enjeu critique : les vulnérabilités web demeurent la principale porte d'entrée des cyberattaques. Les outils d'analyse actuels souffrent toutefois d'un défaut majeur, la multiplication des faux positifs, qui noie les failles critiques sous des alertes non pertinentes.
    
    \medskip
    
    \textit{VulnHunter Pro} a été développé pour résoudre ce problème d'efficacité. S'éloignant des solutions purement statiques, il adopte une architecture hybride : un moteur déterministe assure la fiabilité des détections, tandis qu'un module optionnel d'intelligence artificielle gère l'analyse contextuelle.
    
    \medskip
    
    Cette approche permet de couvrir exhaustivement le référentiel \textbf{OWASP Top 10} tout en garantissant l'exploitabilité des résultats. La version 4.4, validée sur des environnements de test complexes, atteint une précision de 100\% grâce à un mécanisme de déduplication qui élimine toute redondance dans les rapports.
\end{tcolorbox}
\vspace{0.1cm}

\section{Description du projet et objectifs}
\subsection{Objectifs} % <--- FORCE la remontée de la boîte vers le titre

\begin{tcolorbox}[
    colback=green!5!white,
    colframe=green!60!black,
    title=\textbf{\large Objectifs Stratégiques},
    fonttitle=\bfseries,
    rounded corners,
    boxrule=0.8mm,
    drop shadow, % <--- Nécessite l'étape 1 sinon ça plante !
]
    Le projet \textit{VulnHunter Pro} a pour ambition de développer une suite de sécurité offensive complète, capable d'automatiser les tests d'intrusion avec une rigueur professionnelle. Nos objectifs se déclinent selon quatre axes stratégiques :

    \vspace{0.1cm}

    \begin{itemize}[leftmargin=*, label=\textcolor{green!60!black}{$\blacktriangleright$}, itemsep=0.5em]
        
        \item \textbf{Couverture de sécurité maximale :} L'outil doit détecter exhaustivement les vulnérabilités critiques du référentiel \textbf{OWASP Top 10}, incluant les injections (SQL, LDAP, Commandes), les failles XSS, SSRF, et les problèmes de désérialisation. La priorité est donnée à la pertinence des détections pour garantir la sécurité réelle des applications ciblées.

        \item \textbf{Architecture modulaire et évolutive :} Le système est conçu comme une plateforme extensible. La structure modulaire doit permettre l'intégration rapide de nouveaux vecteurs d'attaque (plugins) sans refonte du noyau, assurant ainsi la pérennité de l'outil face à l'émergence de nouvelles menaces.

        \item \textbf{Performance et Précision :} L'efficience est au cœur du développement, avec l'implémentation d'une architecture \textbf{asynchrone} permettant des scans rapides à grande échelle. En parallèle, l'objectif critique est d'atteindre le "zéro faux positif" grâce à des mécanismes de validation et de déduplication avancés.

        \item \textbf{Qualité du Reporting :} Au-delà de la détection, l'outil doit combler le fossé entre la technique et la décision. Les rapports générés doivent être hybrides : intelligibles pour les décideurs (résumés exécutifs, scoring) et immédiatement exploitables par les équipes techniques (preuves de concept, remédiations claires).

    \end{itemize}

\subsection{Fonctionnalités Implémentées}

L'architecture de \textit{VulnHunter Pro} s'articule autour d'un pipeline de \textbf{treize phases distinctes}.

\vspace{0.4cm}

\begin{tcolorbox}[colback=blue!5!white, colframe=blue!75!black, title=\textbf{Phase 1-3 : Reconnaissance et Cartographie}, fonttitle=\bfseries, rounded corners, drop shadow, boxrule=0.5mm]
\begin{itemize}[leftmargin=*, label=\textcolor{blue!70!black}{\tiny$\blacksquare$}, topsep=3pt, itemsep=2pt]
    \item \textbf{Fingerprinting (prise d'empreinte) (\texttt{tech\_detection.py})} : Identification des technologies (serveur, CMS, frameworks, bibliothèques JS).
    \item \textbf{Détection WAF (pare-feu applicatif) (\texttt{waf\_detector.py})} : Identification des Web Application Firewalls (Cloudflare, AWS WAF, ModSecurity).
    \item \textbf{Énumération sous-domaines (\texttt{subdomain\_enum.py})} : Découverte via Certificate Transparency et bruteforce.
    \item \textbf{Scan de ports (\texttt{port\_scanner.py})} : Identification des services exposés.
    \item \textbf{Wayback Machine (\texttt{wayback\_analyzer.py})} : Récupération d'anciennes versions via archive.org.
    \item \textbf{GitHub Recon (\texttt{github\_recon.py})} : Recherche de fuites de code source et secrets.
\end{itemize}
\end{tcolorbox}

\vspace{0.3cm}

\begin{tcolorbox}[colback=red!5!white, colframe=red!75!black, title=\textbf{Phase 4-9 : Moteur de Détection Multi-Vectoriel}, fonttitle=\bfseries, rounded corners, drop shadow, boxrule=0.5mm, breakable]
Le moteur intègre \textbf{29 modules spécialisés} :

\textbf{Injections Critiques :}
\begin{itemize}[leftmargin=*, label=\textcolor{red!70!black}{\tiny$\blacksquare$}, topsep=2pt, itemsep=1pt]
    \item \textbf{SQL Injection} : 50+ payloads (Union, Error, Blind Time/Boolean-based)
    \item \textbf{NoSQL, LDAP, Command Injection}, \textbf{RCE} (exécution de code à distance), \textbf{XXE} (injection XML)
\end{itemize}

\textbf{Failles Client/Serveur :}
\begin{itemize}[leftmargin=*, label=\textcolor{red!70!black}{\tiny$\blacksquare$}, topsep=2pt, itemsep=1pt]
    \item \textbf{XSS} (Reflected, Stored, DOM), \textbf{SSRF}, \textbf{CSRF} (requête inter-site), \textbf{LFI/RFI} (inclusion de fichiers), \textbf{Open Redirect}
\end{itemize}

\textbf{Technologies Modernes :}
\begin{itemize}[leftmargin=*, label=\textcolor{red!70!black}{\tiny$\blacksquare$}, topsep=2pt, itemsep=1pt]
    \item \textbf{GraphQL Fuzzer}, \textbf{API Fuzzer}, \textbf{WebSocket Scanner}, \textbf{Prototype Pollution}, \textbf{Désérialisation}
\end{itemize}

\textbf{Logique Métier :}
\begin{itemize}[leftmargin=*, label=\textcolor{red!70!black}{\tiny$\blacksquare$}, topsep=2pt, itemsep=1pt]
    \item \textbf{Auth Bypass}, \textbf{IDOR} (accès direct non autorisé), \textbf{Business Logic}, \textbf{Config Analyzer}, \textbf{Race Conditions} (conflits de concurrence), \textbf{File Upload}
\end{itemize}
\end{tcolorbox}

\vspace{0.3cm}

\begin{tcolorbox}[colback=purple!5!white, colframe=purple!75!black, title=\textbf{Phase 10-11 : CVE et Menaces 0-Day}, fonttitle=\bfseries, rounded corners, drop shadow, boxrule=0.5mm]

\subsection{Difficultés rencontrées}

\begin{tcolorbox}[colback=red!5!white, colframe=red!75!black, title=\textbf{Défis Techniques et Solutions}, fonttitle=\large\bfseries, breakable]

\textbf{1. Gestion des Faux Positifs}

\textit{Problème :} Taux initial de faux positifs de 77\%. Chaque erreur 500 était interprétée comme vulnérabilité.

\textit{Solution :} Système de validation multicritère :
\begin{itemize}[leftmargin=*, itemsep=0.1em]
    \item Recherche de marqueurs spécifiques (« SQL syntax error »).
    \item Comparaison des réponses normales vs. injectées.
    \item Algorithme de déduplication regroupant les failles similaires.
\end{itemize}
\textit{Résultat :} Passage de 23\% à 100\% de précision.

\vspace{0.3cm}

\textbf{2. Performance Asynchrone}

\textit{Problème :} Race conditions (conflits de concurrence), épuisement des descripteurs de fichiers (« Too many open files »).

\textit{Solution :}
\begin{itemize}[leftmargin=*, itemsep=0.1em]
    \item Sémaphores (verrous de synchronisation via \texttt{asyncio.Semaphore}) limitant les requêtes simultanées.
    \item Pool de connexions TCP via \texttt{aiohttp.TCPConnector}.
    \item Limitation du débit de requêtes avec timeouts adaptatifs.
\end{itemize}

\vspace{0.3cm}

\textbf{3. Intégration des APIs IA}

\textit{Problème :} Quotas, coûts, temps de réponse variable.

\textit{Solution :} Cache pour éviter les appels redondants, IA optionnelle, support de modèles locaux (Ollama).

\vspace{0.3cm}

\textbf{4. Génération de Rapports}

\textit{Solution :} Rapports HTML via Jinja2, conversion PDF optionnelle via outils externes.
\end{tcolorbox}

\subsection{Cahier des charges}
\begin{tcolorbox}[colback=blue!5!white, colframe=blue!75!black, title=Cahier des charges]
Ce projet a été conçu pour atteindre les objectifs suivants :

\begin{itemize}
    \item \textbf{Objectif principal :} Développer un scanner de vulnérabilités web automatisé, capable d’identifier de manière fiable les principales failles de sécurité des applications web, tout en limitant les faux positifs et en proposant un reporting clair et exploitable.
    
    \item \textbf{Public cible :} Étudiants en informatique et en cybersécurité, développeurs web souhaitant évaluer la sécurité de leurs applications, ainsi que professionnels débutants en sécurité offensive, avec une attention particulière portée à la simplicité d’utilisation.
    
    \item \textbf{Fonctionnalités principales :}
    \begin{enumerate}
        \item Phase de reconnaissance de la cible (détection des technologies, énumération des sous-domaines et analyse des ports).
        \item Détection automatisée des vulnérabilités web courantes telles que les injections SQL, les failles XSS, LFI/RFI et SSRF.
        \item Analyse avancée des résultats incluant la modélisation de chaînes d’attaque.
        \item Interface web proposant un tableau de bord pour le suivi des scans et la visualisation des résultats.
        \item Génération de rapports professionnels exploitables.
    \end{enumerate}
    
    \item \textbf{Contraintes techniques :}
    \begin{enumerate}
        \item Le projet doit être développé principalement en Python, en s’appuyant sur une architecture asynchrone afin d’optimiser les performances.
        \item L’outil doit être compatible avec les principaux systèmes d’exploitation (Linux, macOS et Windows via WSL).
        \item Les bibliothèques et frameworks utilisés doivent rester standards et maintenables afin de faciliter l’évolution du projet.
    \end{enumerate}
\end{itemize}

]
Ce manuel décrit les étapes essentielles pour installer, configurer et utiliser \textit{VulnHunter Pro} (v4.4) afin de réaliser des analyses de sécurité web de manière simple, efficace et contrôlée.
\end{tcolorbox}

\vspace{0.5cm}

\textbf{Étape 1 — Installation et configuration}

L’installation du projet s’effectue en clonant le dépôt GitHub puis en créant un environnement virtuel Python afin d’isoler les dépendances. Les bibliothèques nécessaires sont ensuite installées à l’aide du fichier \texttt{requirements.txt}.  
Une configuration optionnelle peut être réalisée via un fichier \texttt{.env} afin de renseigner des clés API utilisées pour activer certaines fonctionnalités avancées, notamment celles liées à l’intelligence artificielle. En l’absence de ces clés, le scanner reste pleinement fonctionnel.

\vspace{0.4cm}

\textbf{Étape 2 — Lancement de l’outil}

Une fois l’installation terminée, l’outil est lancé à l’aide d’un script dédié. L’interface web devient alors accessible depuis un navigateur à l’adresse \texttt{http://localhost:5000}. Cette interface constitue le point d’entrée principal pour l’utilisation de l’outil.

\vspace{0.4cm}

\textbf{Étape 3 — Utilisation du tableau de bord}

Depuis le tableau de bord, l’utilisateur renseigne l’URL cible à analyser et sélectionne les options de scan souhaitées. Le lancement de l’analyse s’effectue en un clic. La progression du scan ainsi que les journaux d’exécution sont affichés en temps réel, permettant un suivi précis du déroulement de l’analyse.

\vspace{0.4cm}

\textbf{Étape 4 — Consultation des résultats}

À l’issue du scan, une synthèse des vulnérabilités détectées est affichée, avec une classification par niveau de gravité. Les résultats peuvent être exportés sous la forme d’un rapport HTML détaillé ou de données brutes au format JSON. Ces rapports incluent les vulnérabilités identifiées, les preuves associées et des recommandations de correction.

\vspace{0.4cm}

\textbf{Étape 5 — Bonnes pratiques d’utilisation}

L’utilisation de \textit{VulnHunter Pro} doit être limitée à des environnements de test ou à des applications pour lesquelles une autorisation explicite a été obtenue. Il est recommandé de commencer par des scans standards avant d’activer des options plus intensives, afin de préserver la stabilité des systèmes analysés.
